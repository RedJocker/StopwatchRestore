Description

Although our stopwatch can run pretty much infinitely, what we really want is to track specific time intervals. Let's set up a stopwatch with a user-defined upper limit that can be changed according to the user's needs.

To add the upper-limit functionality, implement a special pop-up dialog using AlertDialog.

Objectives

  1. Create a new button with the text attribute Settings and the id attribute settingsButton.
  2. The settingsButton is enabled only when the stopwatch is not running.
  3. After the button is clicked, AlertDialog with the following components should appear:
    • EditText with the id attribute upperLimitEditText;
    • Default dialog buttons with the text attributes OK and Cancel.
  4. When importing your AlertDialog don't import the androidx version, it won't work with tests
  5. If there is a number in EditText and the user clicks the OK button, the upper limit is set to that number representing time in seconds. If the user clicks Cancel, the dialog closes.
  6. After the upper limit is reached, the stopwatch color is changed to red (use the RED constant of the class Color). If the number is not set, then there is no upper limit and the stopwatch behaves as in the previous stage.
